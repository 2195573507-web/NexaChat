## Root-Cause And Chain Review

Round 14 moved the app from a local Electron binary shortcut to a verified packaged desktop entry. Before this round, `C:\Users\至亲\Desktop\NexaChat.lnk` launched `node_modules/electron/dist/electron.exe` with the repo root as an argument, and there was no project-owned package script, installer script, shortcut update script, package smoke, or packaged launch manifest.

Implemented chain:

- `DESKTOP_ENTRY` authority -> package scripts -> `release/win-unpacked/NexaChat.exe`.
- Packaged executable -> `resources/app/package.json` -> `dist-electron/main/index.js` -> `nexachat://app/index.html` -> renderer shell.
- Project shortcut script -> packaged executable target -> COM readback -> desktop-entry smoke.
- Startup and crash events -> local `logs/startup.jsonl` under Electron `userData`, with secret/path redaction.
- Local installer script -> isolated installer smoke under `test-results/installer-smoke/<pid>` -> installed executable launch smoke.
