# NexaChat UI Iteration Plan 01

## 计划目标

基于当前截图中的实际界面问题，优先修正显眼的显示瑕疵、信息层级混乱和局部拥挤问题，让界面看起来像一个统一的本地桌面工具，而不是拼接感很强的管理后台。

## 当前问题

- 左侧导航、顶部标题区、页面内容区的间距和对齐不一致。
- 部分页面标题和说明文案过大，导致窄列里出现强制换行、竖排堆叠或截断感。
- 右侧状态栏在每个页面都重复出现，但信息密度偏高，抢占主任务区域的注意力。
- 卡片、按钮、标签、表格的视觉语言不统一，页面看起来偏散。
- 部分区域存在明显空白过多或局部过于拥挤的情况。
- “已实现 / 计划中 / 预留”这类状态的视觉表现不够一致，容易误读。

## 迭代要求

### 1. 页面骨架统一

- 全部模块使用同一套页面骨架：左导航、顶栏、页面标题、模块标签、主内容、可选右侧信息栏。
- 页面标题、模块标签、内容卡片的左边界统一对齐。
- 右侧信息栏保持窄而克制，必要时可在窄窗口优先收起。

### 2. 文本与排版修正

- 页面主标题在默认桌面宽度下保持单行显示。
- 说明文本降级为次级样式，避免在窄卡片里强行撑高版面。
- 表格标题、状态标签和按钮文案控制为可扫描长度，不允许出现影响布局的长串文本。

### 3. 卡片与表格密度统一

- 所有卡片使用统一的圆角、边框、内边距和阴影规则。
- 表格行高、表头、操作列风格统一，不再出现每页一套写法。
- 主操作按钮与次级按钮区分明确，避免同权重堆叠。

### 4. 状态表达统一

- “已实现 / 计划中 / 预留”使用同一状态体系，不靠颜色单独表达。
- 状态标签必须与真实实现状态一致，不允许为了好看制造完成假象。

### 5. 响应式约束

- 在 1280 x 820 与 1040 x 680 下都不能出现文本裁切、控件重叠或主要信息出界。
- 窄宽度下优先收起右侧栏，再收起次级列表区域。

## 验收标准

- 各页面的标题、标签、内容列在视觉上对齐。
- 常规桌面宽度下不再出现明显的文字竖向堆叠、按钮挤压或控件裁切。
- 右侧状态栏仍然可读，但不会压过主工作区。
- 按钮、标签、表格、卡片都呈现一致的视觉系统。
- 页面不会因为某个模块标题过长而破坏整体布局。
- 任何“计划中 / 预留”状态都明确且诚实。

## 风险与非目标

- 本次只做 UI 结构与显示修正，不做大规模品牌重命名。
- 不新增业务功能，只修正展示质量和版式一致性。
- 不依赖数据模型改造来掩盖布局问题。
